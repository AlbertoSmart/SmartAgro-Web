1. Overview 
 
RD-WDCO2 is a gas detection module based on NDIR infrared absorption principle, which is suitable for detecting 
carbon dioxide concentration in aqueous solution. RD-WDCO2 adopts patented optical cavity, imported light source and 
dual-channel detector, which realizes the reference compensation of dual optical paths in space. RD -WDCO2 has the 
characteristics of good selectivity, no oxygen dependence and long life. 
RD-WDCO2 adopts convection diffusion ventilation mode and protective cover, which not only accelerates the 
convection diffusion speed of gas, but also protects the ventilation film. RD -WDCO2 unique waterproof structure, 
removable, easy to clean the sensor outer sleeve. RD -WDCO2 can be used to measure the concentration of dissolved 
CO2 in water, and can also be used to monitor the concentration of CO2 in high temperature and high humidity 
environment. The output signal is optional, which can meet the requirements of different measurement environments and 
various high-precision and high-stability measurements.

8. Sensor Communication Protocol 
 
1. Basic communication parameters: baud rate 9600bps, 8-bit data bit, 1-bit stop bit, no parity bit 
Sensor default address: 32 (20H) 
2. Host Send Protocol Format 
The protocol consists of packets in a fixed format. The size of the packet varies according to the content of the message. 
 
Byte Content 
1 Address of communication unit (sensor address) 
2 Function code 
3 First bit of data area 
4 Second bit of data area 
5 Third bit of data area 
6 Fourth bit of data area 
... Other data 
N-1 Low CRC 
N CRC high position 
 
Address of communication unit: This refers to the address of the lower computer unit when the host communicates with 
the lower computer. The second byte of the packet indicates whether the message is a read command or a write 
command. 03 indicates that the message is a read command, and 06 indicates that the message is a write command. 
CRC is used to verify the correctness of transmitted data. Data is transmitted sequentially from low bytes to high bytes. 
Text is transmitted from left to right. After all data transmission, it will be finished by CRC checking low and high. 
Device return protocol format 
The protocol consists of packets in a fixed format. The size of the packet varies according to the content of the message. 
Command type 
(1) Read the sensor concentration value: For example, read the current 32 (20H) sensor data, 
The host sends commands to the sensor: 
20 03 00 00 00 01 82 BB 
20: Current sensor address 
03: Read the sensor concentration 
00 00: Start register address 
00 01: Number of registers 
82: High CRC 
BB: Low CRC 
The basic principle of cyclic redundancy check code (CRC) is to splice R -bit check code after K-bit information code, and 
the whole code length is N bits, so this code is also called (N, K) code. For a given (N, K) code, it can be proved that 
there exists a polynomial G (x) with the highest power N -K=R. A check code of K bits of information can be generated 
according to G (x), and G (x) is called the generation polynomial of this CRC code. The specific generation process of 
the check code is as follows: assuming that the information to be sent is represented by polynomial C (X), shift C (X) to 
the left by R bit (which can be expressed as C (X) * 2R), so that R bit will be vacated to the right of C (X), which is the 
position of the check code. The remainder obtained by dividing C (x) * 2R by the generated polynomial G (x) is the check

(7) IIC protocol 
The module operates in the slave mode of the IIC and can be connected to an external MCU with internal pull -up 
resistors. The slave address of the module device is: 0x31 
Slave address format: 
BIT7 BIT6 BIT5 BIT4 BIT3 BIT2 BIT1 BIT0 
0 1 1 0 0 0 1 R/W Bit 
R/W Bit: Read = 1/Write = 0 
The address of the module for a write operation is: 0x62 and the address for a read operation is: 0x63. 
Host send timing: 
1）Send start signal 
2）Send address write (slave address + R/W Bit(0) = 0x62) and check answer 
3）Send read command (ASCII 'R': 0x52)) and check for answer 
4）Send stop signal 
5）Send start signal 
6) Send address read (slave address + R/W Bit(1) = 0x63) and check answer 
7）Read 7 bytes from the module and send an answer

9. Considerations 
 
-In environments with EMC problems, shielded protective cables must be used. 
-When the module is cold started, the warm-up time shall not be less than 2 minutes. 
-The sensor should be calibrated periodically, not more than 3 months is recommended. 
-Do not use the sensor for a long time in an environment with high dust density. 
-Please pay attention to the wiring of positive and negative poles of power supply and signal output terminal, and do not 
reverse connection. 
-High concentrations of chemical contamination can cause damage to sensors. Therefore, products should be kept and 
used away from chemical contamination sources. 
-Please refer to the relevant national product installation standards for the installation process.
